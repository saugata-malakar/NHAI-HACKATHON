Datalake 3.0 Integration Steps
1. Copy /src/ml/ and /src/storage/ to Datalake project
2. Add model files to android/assets/models/ and iOS bundle
3. Install 7 npm packages (no native code changes)
4. Wrap EnrollScreen and AuthScreen in Datalake navigator
5. Configure AWS credentials via SyncManager.saveConfig()
6. NetInfo listener auto-syncs when field device goes online
📦  Package Size Impact on Datalake 3.0
7.1 MB model assets + ~2 MB JS code = ~9 MB addition. Well within the 20 MB target. Models are lazy-loaded at startup — no cold-start penalty for users who don&apos;t use facial auth.

The Problem
📵  Zero-Network Zones
Field personnel operate in areas with no internet — standard cloud-based auth completely fails.
🎭  Attendance Fraud
Photo and screen-based spoofing allows fraudulent clock-ins without any real liveness present.
📱  Mid-Range Devices Only
No high-end GPUs. Must run on Snapdragon 6xx / equivalent with 3 GB RAM, Android 8+, iOS 12+.
⚖️  Indian Demographics
Model must handle diverse South Asian skin tones, outdoor glare, shadows, and partial occlusions.
&quot;How can we accurately authenticate field personnel using facial recognition offline, on mid-range devices?&quot;

=== ppt/slides/slide6.xml ===
Face Recognition Pipeline
MobileFaceNet (INT8)
▸ Input: 112×112 RGB, normalized to [-1, 1]
▸ Architecture: Modified MobileNetV2
▸ Depth-wise separable convolutions
▸ Bottleneck residual blocks (×5)
▸ Global Depthwise Conv + Linear
▸ Output: 512-d L2-normalized embedding
▸ Training loss: ArcFace (m=0.5, s=64)
▸ Training data: MS-Celeb-1M + South Asian
▸ INT8 post-training quantization
Cosine Similarity Matching
similarity = Σ(qᵢ · gᵢ)  [both L2-normalized]
Threshold: 0.60  (tunable per deployment)
Enrolled: average of 3 captured embeddings
O(n) gallery scan — loads fully into RAM on auth start

Local Storage Security Stack

🗄️  SQLite Database (op-sqlite)
Face enrollments + auth logs stored in platform sandboxed storage · Not accessible to other apps

🔑  Device-Bound Keys
Embedding key is non-exportable · Cannot be decrypted on any other device even with root access
